## 整理方法

### 1. 自动化脚本
创建了 `organize-docs.sh` 脚本，自动完成以下任务：
- 创建目录结构
- 按规则移动文档
- 生成统计信息
- 创建索引文件

### 2. 分类规则

#### 按文件名模式
- `SYNC_*.md` → `docs/sync/`
- `VOLUNTEER_*.md` → `docs/volunteer/`
- `NETWORK_*.md` → `docs/network/`
- `BACKEND_*.md` → `docs/backend/`
- `FRONTEND_*.md` → `docs/frontend/`
- `*_FIX.md` → `docs/fixes/`
- `*_FEATURE.md` → `docs/features/`
- `*_GUIDE.md` → `docs/guides/`

#### 按功能模块
- 同步相关 → `sync/`
- 义工管理 → `volunteer/`
- 网络配置 → `network/`
- 考勤签到 → `api/checkin/`

#### 按文档类型
- 指南文档 → `guides/`
- 修复记录 → `fixes/`
- 功能特性 → `features/`
- 项目总结 → `summaries/`

### 3. 特殊处理

#### 中文文档
保留原有中文文件名，统一放入对应分类：
- `义工*.md` → `volunteer/`
- `导入*.md` → `volunteer/`
- `批量*.md` → `volunteer/`
- `表格*.md` → `frontend/`
- `排序*.md` → `frontend/`

#### 已存在的文档结构
保留 `docs/api/`、`docs/setup/` 等已有的结构化文档，不做移动。

## 使用指南

### 查看文档索引
```bash
cat docs/README.md
```

### 搜索文档
```bash
# 搜索关键词
grep -r "同步" docs/

# 列出所有文档
find docs/ -name "*.md" -type f

# 按修改时间排序
find docs/ -name "*.md" -type f -printf '%T+ %p\n' | sort -r
```

### 统计信息
```bash
# 统计各目录的文档数量
for dir in docs/*/; do 
  echo "$(basename $dir): $(find $dir -name "*.md" | wc -l) 个文档"
done
```
